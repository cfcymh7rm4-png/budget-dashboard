# GitHub Actions 云端构建部署指南

## 🎯 方案概述

使用 GitHub Actions 在云端完成构建，无需本地电脑配置，推送代码即可自动构建。

```
推送代码 → GitHub Actions 自动构建 → 下载产物 → 上传到妙搭平台
```

## 📋 前置条件

1. 代码已推送到 GitHub 仓库
2. 有 GitHub 账号

## 🚀 使用步骤

### 1. 推送代码到 GitHub

```bash
git add .
git commit -m "更新代码"
git push origin main
```

### 2. 查看构建状态

1. 打开 GitHub 仓库页面
2. 点击顶部 **Actions** 标签
3. 查看最新的工作流运行状态

### 3. 下载构建产物

构建成功后：
1. 点击完成的工作流运行
2. 找到 **Artifacts** 区域
3. 点击 **deploy-package** 下载 `deploy.zip`

### 4. 上传到妙搭平台

1. 打开妙搭平台 → 你的应用 → 部署
2. 选择「上传代码包」
3. 上传下载的 `deploy.zip`
4. 等待部署完成

## ⚙️ 配置说明

### 手动触发构建

除了代码推送自动触发，你也可以手动运行：

1. 进入 GitHub → Actions
2. 选择 **Build and Deploy** 工作流
3. 点击 **Run workflow**

### 内存配置

GitHub Actions 提供 7GB 内存，远超项目所需的 3GB，构建不会内存不足。

## 🔧 故障排查

### 构建失败

1. 点击失败的工作流运行
2. 查看具体步骤的日志
3. 常见原因：
   - 代码语法错误
   - 依赖安装失败
   - 类型检查不通过

### 构建成功但部署失败

1. 检查 `deploy.zip` 是否包含完整内容
2. 确认妙搭平台上传步骤正确
3. 联系妙搭平台支持

## ✅ 优点

- 无需本地配置 Node.js 环境
- 不占用本地电脑内存
- 构建历史可追溯
- 团队协作方便
- 免费（GitHub Actions 免费额度足够）

## ❌ 注意事项

- 每次推送代码都会触发构建
- 构建产物保留 7 天
- 需要手动下载上传到妙搭平台（暂不支持自动部署到妙搭）
